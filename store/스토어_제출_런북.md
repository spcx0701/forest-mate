# 숲길동무 스토어 제출 런북 (Play + App Store)

이 문서대로 따라 하면 **무료 호스팅 배포 → 공개 HTTPS URL → Google Play(AAB) → Apple App Store** 까지 끝난다.
코드/설정은 모두 준비돼 있고, **당신만 할 수 있는 것**(계정·결제·서명키·실기기)은 ⚠️ 로 표시했다.

> 큰 그림: 백엔드가 `app/`(PWA)를 같은 오리진에서 서빙하므로 **배포 1번 = 공개 PWA + API**.
> Android는 그 URL을 TWA로 감싸고(WebView 아님, 진짜 브라우저 엔진), iOS는 Capacitor로 감싼다.

---

## 0. 먼저 정해야 할 값 (한 번만)

| 항목 | 예시 | 메모 |
|------|------|------|
| 패키지/번들 ID | `kr.forestmate.app` | Play·App Store 공통. 출시 후 **변경 불가** |
| 배포 도메인 | `forestmate.onrender.com` | 1단계에서 확정 |
| 팀/개발자명 | 【당신 입력】 | 스토어 표시명 |
| 지원 이메일 | 【당신 입력】 | 심사 필수 |

설정 파일에 박힌 `forestmate.onrender.com` / `kr.forestmate.app` 을 실제 값으로 바꾼다:
`render.yaml` · `app/.well-known/assetlinks.json` · `packaging/android/twa-manifest.json` · `packaging/ios/capacitor.config.json`

## 1. 무료 호스팅 배포 (공개 HTTPS URL 만들기) — 스토어의 토대

### 추천: Render (Docker, 무료, 카드 등록 불필요)
1. ⚠️ 코드를 GitHub 저장소에 push (`forest-mate/`가 루트인 repo).
2. ⚠️ https://render.com 가입 → **New → Blueprint** → 저장소 선택.
   `render.yaml`을 자동 인식해 `forestmate` 웹 서비스를 만든다.
3. 배포 완료 후 URL 확인: `https://forestmate.onrender.com`
   - 랜딩 `/home.html` · 앱 `/index.html` · 관제 `/dashboard.html` · API 문서 `/docs`
4. (선택) ⚠️ 대시보드 → Environment 에서 비밀키 입력:
   `DATA_GO_KR_KEY`(공공데이터포털), `ANTHROPIC_API_KEY`(Claude). 없으면 스냅샷·규칙 폴백.

> 무료 플랜은 15분 유휴 시 슬립 → 첫 요청에 30~50초 콜드스타트. 심사·시연 직전 한 번 깨워두면 된다.
> 데이터 영속이 필요하면 `render.yaml`의 PostgreSQL 블록 주석을 해제(무료 DB).

### 대안
- **Railway** (`railway up`, Dockerfile 자동 인식, 월 $5 크레딧 무료) — 슬립 없음.
- **Fly.io** (`fly launch`, 무료 한도) — 한국 리전(nrt) 가능.
- **PWA만 따로**(API 없이 정적): Cloudflare Pages / Netlify / Vercel 에 `app/` 폴더 배포. 단 이 경우 cloud 모드(LIVE)·관제 실시간은 빠지고 로컬 폴백으로만 동작.

✅ **검증**: 브라우저로 `https<도메인>/index.html` 접속 → 설치 배너(⊕) 표시, `/.well-known/assetlinks.json` 200, `/api/v1/healthz` 200 이면 성공.

## 2. Google Play — TWA(AAB) 빌드 & 등록

### 2-1. 사전 준비 ⚠️
- **Google Play Console 개발자 계정**($25 1회 결제, 본인 인증 1~2일).
- 로컬에 **JDK 17+** 와 **Node 18+**. (이 Mac엔 Java 미설치 → `brew install openjdk@17`)

### 2-2. AAB 빌드 (Bubblewrap)
```bash
npm install -g @bubblewrap/cli
cd forest-mate/packaging/android

# 배포한 manifest URL로 초기화 (twa-manifest.json을 덮어쓴다)
bubblewrap init --manifest https://forestmate.onrender.com/manifest.json
#  → packageId: kr.forestmate.app, 서명키 생성 여부 yes (비밀번호 메모!)

bubblewrap build
#  → app-release-bundle.aab (Play 업로드용) + app-release-signed.apk (실기기 테스트용)
```

### 2-3. Digital Asset Links 연결 (필수 — 안 하면 주소창이 보임)
1. 서명키 SHA-256 지문 확인:
   ```bash
   keytool -list -v -keystore android-signing.keystore -alias forestmate
   ```
   또는 Play Console → 앱 무결성 → **앱 서명 키 인증서**의 SHA-256.
2. 그 지문을 `app/.well-known/assetlinks.json` 의 `REPLACE_WITH_...` 자리에 넣고 **재배포(1단계)**.
   > Play **앱 서명(Play App Signing)**을 쓰면 Google이 최종 서명키를 관리하므로, Console에 표시된
   > **앱 서명 키**의 SHA-256을 넣어야 한다(업로드 키 아님). 둘 다 넣어도 무방.
3. 검증: `https://<도메인>/.well-known/assetlinks.json` 이 그 지문을 반환하는지 확인.

### 2-4. Play Console 등록 ⚠️
- 앱 만들기 → 프로덕션 트랙에 `app-release-bundle.aab` 업로드.
- 스토어 등록정보: `store/스토어_등록정보.md`의 제목·설명·그래픽 그대로 사용.
- 스크린샷: `app/screens/home.png · trail.png · ai.png · sos.png`(폰), `dashboard.png`(태블릿/소개용).
- **데이터 보안(Data safety)**: 위치(정밀)·건강(선택) 수집, 전송 중 암호화, 삭제 요청 가능, 백그라운드 위치 사유 = "산행 중 안전 안내·조난 감지". (`스토어_등록정보.md` 요약 사용)
- 개인정보처리방침 URL: `https://<도메인>/legal/...` 또는 별도 호스팅(4단계).
- 콘텐츠 등급 설문, 타깃 연령, 광고 없음 체크 → 심사 제출(보통 수시간~3일).

## 3. Apple App Store — Capacitor 래퍼

> ⚠️ iOS 빌드는 **macOS + Xcode** 필요(이 Mac 가능). **Apple Developer Program**($99/년) 가입 필수.
> Apple 심사 4.2(최소 기능) 대비: 단순 웹뷰가 아니라 **네이티브 위치·푸시·카메라**를 쓰는 안전 앱임을 강조한다.

### 3-1. 프로젝트 생성
```bash
cd forest-mate/packaging/ios
npm install
npm run sync          # app/ → www 복사 + iOS 프로젝트 동기화
npx cap add ios
npx cap open ios      # Xcode 실행
```
`capacitor.config.json`은 `server.url`로 배포 URL을 직접 로드한다(=cloud 모드 유지).
오프라인 번들을 원하면 `server.url`을 지우고 `app/index.html` 상단에
`<script>window.FM_API_BASE="https://<도메인>/api/v1"</script>` 를 추가한 뒤 다시 sync.

### 3-2. Xcode 설정 ⚠️
- Signing & Capabilities → 본인 Team 선택, Bundle ID `kr.forestmate.app`.
- Capabilities 추가: **Background Modes**(Location updates), **Push Notifications**.
- `Info.plist` 사용 목적 문자열(심사 거부 1순위 — 반드시):
  - `NSLocationWhenInUseUsageDescription` = "산행 중 실시간 안전 안내와 위험 구간 경고를 위해 위치를 사용합니다."
  - `NSLocationAlwaysAndWhenInUseUsageDescription` = "앱이 백그라운드일 때도 조난 감지·긴급구조 연계를 위해 위치를 사용합니다."
  - `NSCameraUsageDescription` = "독버섯·독성 식물 식별을 위해 카메라를 사용합니다."

### 3-3. 제출 ⚠️
- Xcode → Product → Archive → Distribute → App Store Connect 업로드.
- App Store Connect: 앱 생성, 스크린샷(6.7"·6.5"·5.5" 또는 1290×2796 등 규격 필요 — `app/screens/*` 리사이즈),
  설명(`스토어_등록정보.md`), 개인정보 URL, App Privacy 설문(위치·건강), 심사 메모에
  "산림청 공공데이터 기반 산행 안전, 백그라운드 위치=조난 감지 목적" 기재 → 제출(심사 1~3일).

## 4. 법무 문서 호스팅 (양 스토어 필수)

개인정보처리방침·이용약관 **공개 URL**이 있어야 심사를 통과한다. 가장 쉬운 방법:
- 이미 배포한 서버가 `legal/`을 서빙하도록 하거나(현재 `.dockerignore`가 `legal/` 제외 → 포함하려면 제외 줄 삭제 후 재배포),
- 또는 GitHub Pages / Notion 공개 페이지에 `legal/개인정보처리방침.md`·`legal/이용약관.md` 내용을 올리고 그 URL 사용.
- `legal/` 문서의 【팀명】·【이메일】·【개인정보 보호책임자】 자리표시자를 먼저 채울 것.

## 부록 — 내가(코드) 끝낸 것 vs 당신이 해야 하는 것

| 끝남(코드/설정) | 당신 ⚠️(계정·결제·실기기) |
|---|---|
| PWA 설치형 정비(manifest·아이콘·스크린샷·shortcuts) | GitHub push, Render 가입·배포 |
| `assetlinks.json` 서빙 라우트 + 템플릿 | 서명키 생성 → SHA-256 채우기 |
| Render 무료 배포 청사진(`render.yaml`) | Play Console($25)·Apple Developer($99) 가입 |
| TWA 설정(`twa-manifest.json`) | `bubblewrap build`(JDK 설치) → AAB |
| iOS Capacitor 설정·스크립트 | Xcode Archive → App Store Connect 제출 |
| 스토어 등록 메타데이터·법무 문서 초안 | 자리표시자 채우기 + 법무 URL 호스팅 + 심사 제출 |
