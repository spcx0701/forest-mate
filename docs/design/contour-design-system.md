# 등고선 디자인 시스템 (Contour Line Design System)

Cross-platform source of truth for ForestMate / 숲길동무, ported from the Claude
Design file **`ForestMate System.dc.html` → system 1 · 등고선 / CONTOUR**. The
native Kotlin phone and Wear apps lost the app's visual language during the
Kotlin port; this system restores and unifies it under one topographic visual
language shared by web, the Kotlin phone app, and the Kotlin Wear app.

> The design defines three "elevations" of the same identity. **System 1 ·
> 등고선/CONTOUR** (this doc) is the mobile default and the implemented target.
> System 2 (야간/NIGHT TACTICAL — neon contour on black, B2G dashboard) and
> system 3 (능선/RIDGELINE — green→blue gradient mesh + glass, for onboarding/AI)
> reuse the same tokens at higher expression density.

## 1. Concept

"종이 지도와 측량 도면의 미학." Paper-map and survey-drawing aesthetic: **cream
paper** ground, **ink-green living contour texture**, and the **green→teal→blue
gradient used only for "what matters now"** (index ring, course hero, CTAs, SOS).
Core identity held constant across platforms: 숲이 mascot · green→blue gradient ·
contour motif · Korean typography.

## 3. Typography

- Display & numerals: **Space Grotesk** (700/800) — scores, distances, stats.
- Indicators: **JetBrains Mono** (500/600) — coordinates (국가지점번호 / N·E),
  time, indices, metric rows. letter-spacing .06–.14em, often uppercase.
- Korean body/headings: **Pretendard** (600–800).
- Eyebrow: Space Grotesk 600, letter-spacing .12–.2em, uppercase, teal.

Native apps approximate Space Grotesk with `sans-serif-black/medium` and JetBrains
Mono with the platform monospace until the brand TTFs are bundled. Web uses the
font names with monospace/system fallbacks (offline-safe).

## 4. Spacing, radius, elevation

Spacing step 4 (6/8/12/14/18). Radius: card 18–22 · pill 999 · tile 8–14 ·
phone screen 38. Shadow `0 6px 16px -10px rgba(40,60,40,.3)`. Screen gutter 16.

## 5. Contour motif (the signature)

Generated by the design's `ring()` algorithm — **organic, irregular** concentric
ridge lines, not circles:

```
r = baseR + amp·sin(3a + seed) + 0.45·amp·cos(5a + 1.4·seed)
x = cx + cos(a)·r ;  y = cy + sin(a)·r·0.82   // vertical squash
baseR = 16 + i·step ;  seed = i·1.7 ;  stroke 1.1
```

Named fields from the design:
- `cInk` — `#163E2A` op .13, n15 step17 amp15 (cream backgrounds)
- `cInkSoft` — `#163E2A` op .08 (large soft fields)
- `cGhost` — `#ffffff` op .18, n14 step16 amp13 (dark/gradient hero overlays)

Implementations: web `app/contour-ink.svg` + `app/contour-ghost.svg` (generated
from the same algorithm); Android `ui/ContourBackground.kt` (ported generator);
Wear `WatchFaceView.buildContourRing` (inlined generator).

## 7. Feature inventory (pre-Kotlin parity — must remain)

Behavior source of truth is the web app (`app/app.js`); the Kotlin apps must keep
all of it while re-skinned:

- **Home:** 산행지수 ring + region, 2×2 sub-index (산불/기상/강수/바람), AI 맞춤
  코스 cards (match %), nationwide mountain search, local↔cloud fallback.
- **Course detail:** hero, elevation, metadata (peak/거리/시간/난이도), hazards,
  start map, external directions (Kakao/Google), 국가지점번호/GPS.
- **Hike:** OSM/Leaflet map, route + GPS track, start/summit/hazard/rescue/current
  markers, grid tag, progress + stats, hazard alerts, start/pause/end, GPS sync,
  **watch pairing**.
- **SOS:** hold-to-confirm, grid number, GPS, rescue station, server `/sos`.
- **AI 숲이:** chat, quick prompts, danger flag + confidence, source/engine labels.
- **My:** account register/login (+social), summary, hike log, calendar, badges.
- **Watch:** 6-digit backup pairing, compact face (route/progress/HR/GPS/battery/
  distress), foreground sensor upload.

## 8. Platform mapping

- **Web** (`app/`): `app/contour.css` + generated SVGs layered over `app.css`.
- **Kotlin phone** (`packaging/android/app`): `ui/ContourTheme.kt` (tokens +
  drawables), `ui/ContourBackground.kt` (contour field), `ui/NativeViews.kt`
  (component kit incl. `ScoreRingView`, `BrandMarkView`), `MainActivity.kt`.
- **Kotlin Wear** (`packaging/android/wear`): `WatchFaceView` cream face + organic
  contour + green→blue ring; themed pairing controls; `res/values/colors.xml`.

## 9. Known approximations

- Brand fonts (Space Grotesk / JetBrains Mono) are not yet bundled as Android
  font resources or web-font files (offline constraint) — platform fallbacks used.
- Native apps render system 1 (CONTOUR) for all screens; systems 2–3 (night/
  ridgeline) are documented but not separately themed yet.
